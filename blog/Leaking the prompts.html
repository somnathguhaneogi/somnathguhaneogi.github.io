Public LLMs have become standard in every red team and pentest workflow. They help craft payloads, explain complex vulns, generate reports, and even debug code in seconds. But they are also the fastest route to an accidental data breach that can end careers and engagements.

Every week we hear of new incidents: consultants pasting real JWTs, API keys, Burp exports, internal screenshots, and live endpoints into ChatGPT, Gemini, Claude or Grok. The data leaves your machine forever — stored for training, visible to support staff, or later regurgitated by the model.

This guide gives you the complete, battle-tested 10-rule framework used by professional red teams to use public AI without ever risking client data.

// RULE SET v1.0
AI-01 CRITICAL
Rule 01 – Never paste real secrets, tokens, keys, passwords or credentials

One single JWT, API key, or password in any public LLM = material data breach under GDPR / CCPA.

The moment you hit send, the secret is logged, stored for 30–90 days (or longer), and in some cases added to training data. Even if you delete the chat, the provider may still retain it. Attackers have already extracted real credentials from public model training dumps.

Real case (2024): A senior pentester at a Big-4 firm pasted a client’s production API key into ChatGPT to “debug rate limiting”. The key later appeared in a public training data leak. Client suffered fraud losses and sued. The consultant was terminated and the firm paid $2.3M settlement.

Safe alternative: Use dummy values like “eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.dummy…” or describe the structure only.

AI-02 CRITICAL
Rule 02 – Never upload screenshots containing real UI, domains, internal data or error messages

Even blurred or cropped images can be reconstructed by modern vision models.

Modern LLMs with vision (GPT-4o, Claude-3.5, Gemini 1.5) can read text from screenshots. Internal hostnames, GUIDs, error stacks, and even blurred tokens are often recoverable.

🔴
Real case (2025): A red teamer uploaded a blurred Burp screenshot of an internal banking app. The model correctly extracted the internal domain and a partial JWT. Client’s OSINT team discovered their own subdomain in public AI conversation archives.

Safe: Use text descriptions or create clean mockups in draw.io / Excalidraw.

AI-03 HIGH
Rule 03 – Always fully anonymize / pseudonymize EVERY piece of data

Replace real domains, IPs, usernames, company names, and internal IDs before sending.

Even “example.com” can be dangerous if you include enough context. Real-life mistake: pentesters replaced “bank.com” with “example.com” but left the exact path and parameter names — enough for the client to identify themselves during a later AI audit.

⚠️
Real case (2024): Consultant used “acme-bank.internal” in a prompt. The model later suggested payloads using that exact string. Client’s security team found the conversation via their own brand monitoring and terminated the engagement.
AI-04 HIGH
Rule 04 – Only use enterprise / private / zero-retention instances (or local models)

Free/public tiers log everything for training. Enterprise plans with zero data retention are the minimum.

Samsung’s 2023 incident started exactly here: employees using the free ChatGPT tier. The company banned it after source code and meeting notes were found in training data.

⚠️
Real case (2023-2024): Samsung banned ChatGPT company-wide after three separate incidents where employees pasted proprietary code and internal meeting notes.
AI-05 HIGH
Rule 05 – Never ask AI to write full exploit code for real in-scope assets

Describe the vulnerability class generically — never include live URLs, parameters, or tokens.

Real mistake: Pentester asked “Write a Python exploit for this exact endpoint https://target.com/api/v3/admin?user=...” — the model generated working code that was later found in public GitHub repos indexed by the client.

AI-06 MEDIUM
Rule 06 – Never generate final reports, findings, or deliverables with public AI

Clients now run AI detectors. Perceived lack of professionalism can kill future contracts.

⚠️
Real case (2025): A boutique pentest firm submitted an AI-generated report. Client ran it through multiple detectors, questioned the entire engagement, and refused payment.
AI-07 HIGH
Rule 07 – Delete every conversation immediately after use

Even “temporary chat” modes are stored for weeks. Shared accounts multiply the risk.

AI-08 HIGH
Rule 08 – Prefer fully local / offline models for anything even remotely sensitive

Ollama + Llama 3.1 70B, Qwen2.5-Coder, DeepSeek-Coder-V2, Gemma 2, or Phi-4 — zero data ever leaves your laptop.

AI-09 HIGH
Rule 09 – Never feed live endpoints, real HTTP responses, or full Burp captures

Even 403/401 responses often contain internal server headers, stack traces, GUIDs, and version info.

AI-10 HIGH
Rule 10 – Always document AI usage and obtain explicit client approval in the SOW / RoE

Add a one-line clause: “Limited use of anonymized queries against approved enterprise AI instances with zero data retention”.

✅
Most professional firms now include this exact clause. It protects both you and the client.
